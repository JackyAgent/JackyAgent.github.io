用 Go 构建一个高并发任务调度器

2026-06-18 · 12 分钟阅读 · 杨佳琦

背景

调度器是后端系统的核心组件，负责把大量异步任务分发给有限的 worker。一个简单的实现很容易在千级 QPS 下崩溃——热点往往出在全局锁与任务队列的拷贝上。

func (s *Scheduler) Schedule(job Job) error {
    shard := uint64(job.ID) % s.shardCount
    return s.shards[shard].Push(job)
}

锁竞争优化

我们的做法是把全局队列拆成 N 个分片队列，每个 worker 固定消费一个分片，并用无锁 ring buffer 承载入队。压力下再把分片数动态上调，热点被均匀打散。

“过早优化是万恶之源，但调度器的热点必须一开始就量化。”
